Process Model Report

The process model report offers static information derived from the model definition.

Stardust offers a pre-installed BIRT version to be able to run the process model report. If you use your own BIRT installation, please refer to the corresponding Birt Installation Guide on how to install BIRT. Refer to the Stardust Release Notes for details on the supported Birt version.

Start the Process Model Report

To run the process model report:

  • In the main toolbar select the Process Model Report button.
  • Choose the report format HTML or PDF.
  • OK the dialog.


Figure: Starting Model Process Report

Definitions

Now the Report Viewer opens with details on your model. To view these details there are several definition views:

  • Overview
  • Process Definition Details
  • Application Details
  • Role Details
  • Organization Details
  • Activity Details
  • Data Details

Definition - Overview

The report Definition - Overview gives an overview of the model's basic attributes like name, description and id as well as the following components:

  • Process Definitions
  • Roles
  • Organizations
  • Applications
  • Data

The name and the description of each component are displayed. Click the Details link in the Definition column to go to the detailed definition report for each component.

Definition - Process Definition Details

A Process Definition Details report can be opened by clicking on a Details link behind the name of a process definition in the Definition Overview report. It shows detailed information about a process definition. In addition to the name, the description and the id of the selected process definition, all model components directly related to it are listed in this report, like:

  • Starting Roles or Organizations
  • Activities
  • Subprocesses
  • Superprocesses

Starting Roles or Organizations

This section shows a list of all roles or organizations which are entitled to create and start a new instance of the process definition. The names displayed link to the detailed Report for the roles or organizations.

Activities

All activities are listed, which are part of the process definition and therefore can become part of its instances. A table containing description, performer and application used, is displayed for each of these activities. Click on the names of the activities or components to go to the detailed report on them.

Subprocesses and Superprocesses

This table shows all sub- and superprocesses of the selected process definition. Click their names to link to their respective Process Definition Detail report.

Definition - Application Details

An Application Details report can be opened by clicking on the Details link behind the name of an application in the Definition Overview report. It shows information about an application, like name and description.

Definition - Role Details

A Role Details sample report can be opened by clicking on the Details link behind the name of a role in the Definition Overview report. It shows detailed information about a role, like name, description and activities.

Definition - Organization Details

An Organization Details sample report can be opened by clicking on the Details link behind the name of an organization in the Definition Overview report. It shows detailed information about an organization, like name, description and activities.

Definition - Activity Details

An Activity Details report can be opened by clicking the name of an activity in a Process Definition Details report. It shows detailed information about an activity. In addition to the type, name, description and id of the selected activity, all model components directly related to it are listed in this report:

  • Performer
  • Application
  • Model Name
  • Process Definition Name
  • Preceeding Activities
  • Succeeding Activities
  • Input Data
  • Output Data

Performer

This section displays the organization or role performing this activity.

Application

Application used to perform this activity.

Preceding Activities

In this section the names of all activities directly preceding this activity in its process definition are listed. The activity names link to their respective Activity Detail reports.

Succeeding Activities

In this section the names of all activities directly succeeding this activity in its process definition are listed. The activity names link to their respective Activity Detail reports.

Input Data

This table lists the names, types, Ids and descriptions of all data objects which may serve as input for this activity.

Output Data

This table lists the names, types, Ids and descriptions of all data objects which may be altered during this activity.

Definition - Data Details

This section displays the name, id, type and description of the selected data. Additionally there are two tables to list the activities, which the data is input and output for. Clicking on the activities names link to their Details definition.
