Use Case - Using Declarative Security for Scoped Participants in Report Viewing

This tutorial describes a simple use case to demonstrate how declarative security set for reading activity instance data affects the visibility of the reporting data for users in different departments.

Contents

  • Creating the Model and Organizational Structure
  • Creating the Users and Departments in the Stardust Portal
  • Viewing the Results in the Predefined Report Monitoring Overview

You can create a new Stardust model or import the example model DemoReportingModel.xpdl provided by Stardust by extracting it from the scopedParticipants/dat folder residing in the following ZIP file:

scopedParticipants.zip

In case you import the model, you can skip the chapter Creating the Model and Organizational Structure and continue with chapter Creating the Users and Departments in the Stardust Portal.
